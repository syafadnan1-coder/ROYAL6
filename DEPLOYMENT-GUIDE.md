# 🟢 المرحلة 1: ربط Supabase

## الخطوة 1.1: إنشاء حساب Supabase (مجاني)

1. ادخل: **https://supabase.com**
2. اضغط **"Start your project"**
3. سجّل عبر GitHub أو البريد الإلكتروني
4. أكّد بريدك من الرسالة التي تصلك

## الخطوة 1.2: إنشاء مشروع جديد

1. اضغط **"New Project"**
2. اختر Organization (الافتراضي مناسب)
3. املأ:
   - **Name**: `royal-pipes`
   - **Database Password**: اختر كلمة قوية واحفظها!
   - **Region**: اختر `Frankfurt (eu-central-1)` أو `Bahrain` (الأقرب لليمن)
4. اضغط **"Create new project"**
5. انتظر دقيقتين حتى يكتمل الإنشاء

## الخطوة 1.3: إنشاء الجداول (الأهم!)

1. من القائمة اليسرى → اضغط **"SQL Editor"**
2. اضغط **"+ New Query"**
3. **اذهب لموقعك** → لوحة التحكم → ⚡ ربط السحابة → اضغط "📋 عرض كود SQL"
4. اضغط زر **"نسخ الكود"** الأخضر
5. ارجع لـ Supabase → الصق الكود في SQL Editor
6. اضغط زر **"RUN"** في الأعلى
7. ✅ سترى رسالة "Success" + ستظهر 5 جداول + 11 منتج + 4 وكلاء جاهزة

## الخطوة 1.4: الحصول على API Keys

1. من القائمة اليسرى → اضغط **⚙️ Settings**
2. اضغط **API**
3. ستجد:
   - **Project URL** (مثل: `https://xxxxx.supabase.co`)
   - **anon / public** key (مفتاح طويل يبدأ بـ `eyJ...`)
4. **انسخهما** واحتفظ بهما

## الخطوة 1.5: إنشاء Bucket للصور (لرفع صور المنتجات)

1. من القائمة اليسرى → اضغط **Storage**
2. اضغط **"New bucket"**
3. الاسم: **`products`** (مهم - بنفس الكتابة)
4. ✅ فعّل **"Public bucket"** (عام)
5. اضغط **"Save"**
6. على الـ bucket الذي أنشأته → اضغط **Policies**
7. اضغط **"New Policy"** → **"For full customization"**
8. أدخل:
   - **Policy name**: `Allow public uploads`
   - **Allowed operation**: ✅ SELECT, ✅ INSERT
   - **Target roles**: `public`
   - **USING expression**: `true`
   - **WITH CHECK expression**: `true`
9. اضغط **Review** → **Save policy**

## الخطوة 1.6: ربط الموقع بـ Supabase

1. ادخل موقعك → **لوحة التحكم** (admin / admin123)
2. اضغط تبويب **"⚡ ربط السحابة"** (الأعلى - أصفر نابض)
3. الصق **Project URL** في الخانة الأولى
4. الصق **anon key** في الخانة الثانية
5. اضغط الزر الأخضر **"☁️ تفعيل السحابة"**
6. ✅ ستظهر رسالة: "☁️🎉 تم تفعيل السحابة بنجاح!"

# 🟢 المرحلة 2: نشر الموقع على Netlify (مجاني)

## لماذا Netlify؟
- ✅ مجاني تماماً مدى الحياة
- ✅ نطاق مجاني (مثل: `royal-pipes.netlify.app`)
- ✅ SSL/HTTPS تلقائي
- ✅ سرعة عالية جداً
- ✅ نشر بسحب وإفلات!

## الطريقة الأسهل: السحب والإفلات

### الخطوة 2.1: بناء الموقع للنشر

في حاسوبك، افتح Terminal أو CMD داخل مجلد المشروع، ونفّذ:

```bash
npm install
npm run build
```

سيظهر لك مجلد جديد اسمه `dist/` يحوي ملف `index.html`

### الخطوة 2.2: رفع الموقع لـ Netlify

1. ادخل: **https://app.netlify.com/drop**
2. سيظهر صندوق كبير في وسط الصفحة
3. **اسحب وأفلت مجلد `dist`** كاملاً في الصندوق
4. ✅ انتظر 30 ثانية... ومبروك! الموقع رُفع!
5. ستحصل على رابط مثل: `https://wonderful-pipes-12345.netlify.app`

### الخطوة 2.3: (اختياري) تغيير اسم الموقع

1. في Netlify، اضغط على موقعك
2. **Site settings** → **Change site name**
3. اكتب: `royal-pipes` → سيصبح: `https://royal-pipes.netlify.app`

### الخطوة 2.4: (اختياري) ربط نطاق خاص

1. اشترِ نطاقاً من Namecheap/GoDaddy
2. في Netlify → **Domain settings** → **Add custom domain**
3. اتبع التعليمات

# 🟢 المرحلة 3: اختبار التزامن

## الاختبار النهائي (دليل أن كل شيء يعمل):

### الخطوة 3.1: إضافة منتج تجريبي
1. ادخل موقعك على Netlify (مثلاً: `https://royal-pipes.netlify.app`)
2. اضغط **لوحة التحكم** → سجّل بـ `admin` / `admin123`
3. **مهم**: تأكد أن **⚡ ربط السحابة** ما زالت مفعّلة (انتقل لتبويب السحابة وادخل البيانات مرة أخرى إذا لزم)
4. اذهب لـ **المنتجات** → **إضافة منتج جديد**
5. املأ البيانات + ارفع صورة من جهازك
6. اضغط **حفظ**
7. ✅ ستظهر رسالة: **"☁️✓ تم نشر المنتج للسحابة! العملاء في كل اليمن يرونه الآن 🎉"**

### الخطوة 3.2: التحقق أن العملاء يرونه

#### الطريقة 1: من نفس الجهاز
1. افتح المتصفح في **وضع التصفح الخفي** (Ctrl+Shift+N أو Cmd+Shift+N)
2. ادخل موقعك مرة أخرى
3. ✅ يجب أن ترى المنتج الجديد

#### الطريقة 2: من جوالك
1. افتح المتصفح في جوالك
2. ادخل رابط موقعك
3. ✅ يجب أن ترى نفس المنتج الجديد

#### الطريقة 3: من حاسوب صديق
1. اطلب من صديق يفتح الرابط
2. ✅ يرى نفس المنتج

# ⚠️ ملاحظات مهمة جداً:

### 1. تفعيل السحابة من نفس الجهاز
بيانات Supabase تُحفظ في متصفحك الخاص بلوحة التحكم. لذلك:
- ✅ في كل جهاز/متصفح تستخدمه للإدارة، يجب أن تدخل ⚡ ربط السحابة مرة واحدة
- ✅ بمجرد إدخال البيانات، تبقى محفوظة حتى لو أغلقت المتصفح

### 2. الزوار (العملاء) لا يحتاجون إعدادات
العملاء يرون كل شيء تلقائياً - فقط لوحة التحكم تحتاج تفعيل السحابة لإضافة المنتجات.

### 3. الصور تُرفع لـ Supabase Storage
عند رفع صورة منتج من جهازك → تُرفع لـ Supabase → يحصل المنتج على رابط دائم

### 4. السحابة = مفتاح كل شيء
- ❌ بدون تفعيل السحابة → المنتجات تُحفظ محلياً في متصفحك فقط
- ✅ مع تفعيل السحابة → تُحفظ في قاعدة بيانات حقيقية وتظهر للعالم
